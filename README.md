**Contexte :** Robot agricole, parsing données température/humidité  
**Problème :** Fuites mémoire, buffer overflow, pas de gestion erreur

### 2. TypeScript - API Parcelles (web)

**Contexte :** Dashboard agricole temps réel  
**Problème :** Typage faible, XSS, pas de cache, pas de gestion erreur  
[Voir l'exercice](./typescript-api/)

## Quick Start

1. Clone ce repo
2. Choisis ton exercice (C++ ou TypeScript)
3. Identifie
4. Tu as le droit d'exécuter **un seul** prompt avec github copilot pour améliorer le code, sans code à la main
5. Créé une branche sur la qule tu vas push le prompt exécuté ainsi que le code amélioré

## Objectifs pédagogiques

- Identifier les problèmes dans du code legacy
- Structurer un prompt efficace pour l'IA
- Obtenir du code production-ready du premier coup
